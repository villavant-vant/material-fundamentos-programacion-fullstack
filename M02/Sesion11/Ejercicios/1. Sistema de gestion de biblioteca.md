# Ejercicio 1 (mini-proyecto integrador): Sistema de Gestión de Biblioteca

Proyecto de consola, sin persistencia, que integra todo lo visto en el módulo (encapsulamiento, relaciones, herencia y ArrayList).

Ya cuentas con la base en [Clases](../Clases): `CLibro` y `CBiblioteca` (composición: la biblioteca tiene un `ArrayList<CLibro>`).

Completa el sistema agregando:

1. Clase `Usuario` (clase padre) con atributos comunes (nombre, id) y método `mostrarInfo()`.
2. Subclases `Administrador` y `Lector` que hereden de `Usuario` y sobrescriban (`@Override`) `mostrarInfo()`.
3. Clase `Prestamo` que relacione un `CLibro` con un `Usuario` (y opcionalmente una fecha).
4. La clase `CBiblioteca` debe tener también un `ArrayList<Usuario>` y un `ArrayList<Prestamo>`, con métodos para registrar préstamos y devoluciones.
5. Interfaz `Imprimible` con el método `imprimirResumen()`, implementada por `CBiblioteca` (resumen de libros, usuarios y préstamos activos).

Actividades sugeridas:
- Arma primero el diagrama de clases (UML simple) antes de programar.
- Organiza el código en paquetes: `model`, `service`, `main`.
- Desde `main`, crea usuarios de ambos tipos, registra préstamos y muestra el resumen usando polimorfismo (`mostrarInfo()` de cada usuario).
